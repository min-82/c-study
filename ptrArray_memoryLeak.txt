1. 메모리 누수(Memory Leak) 방지 원칙

    현상: 포인터 변수에 새 주소를 대입하면, 기존에 가리키던 주소값을 잃어버려 해당 메모리 공간이 '미아'가 됨.

    해결책 1 (백업): 주소를 바꾸기 전, 다른 포인터 변수에 기존 주소를 복사해 둠. (old_ptr = ptr;)

    해결책 2 (해제): 기존 데이터가 더 이상 필요 없다면 새 주소를 받기 전에 반드시 free(ptr);를 실행함.

2. 포인터 배열 (Array of Pointers)

    구조: 실제 데이터가 아닌, 데이터가 있는 '주소값'들을 모아놓은 배열.

    메모리 배치:

        주소값들은 배열이므로 메모리에 연속적으로 붙어 있음.

        실제 데이터(구조체 등)는 malloc으로 각각 할당받았으므로 메모리 여기저기 흩어져 있음.

    장점: 데이터 덩어리를 직접 옮기지 않고 8바이트 주소값만 교체하면 되므로, 데이터 정렬이나 순서 변경이 매우 빠르고 효율적임.
